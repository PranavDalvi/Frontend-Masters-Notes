1. **When designing a video upload system, why is it beneficial to separate metadata storage from video blob storage?**
	1. Blob storage is more optimized for video and audio files, while databases are better suited for text data like metadata. Additionally, keeping the system stateless by writing metadata to a database prevents problems where uploads complete but have no associated metadata, which would create orphaned files that are useless and cannot be properly managed or deleted.
2. **In a video streaming application, what is the primary purpose of a notification service in the upload workflow?**
	1. The notification service watches for when video uploads are complete to the source video storage, notifies the client that the upload is done, and triggers the processing service to begin processing the video. It acts as a central component that coordinates communication between different parts of the system.
3. **Why is it important to understand the user flow before diving into technical implementation details when designing a system?**
	1. Understanding the user flow helps identify what the actual problems and bottlenecks will be in the system. Without understanding the domain and workflow first, you risk overdesigning one particular part while missing critical components, or choosing optimizations that don't address the real constraints of the system.
4. **When designing the API for a video upload system with the endpoint `POST /videos`, what are the key components that need to be included in the request?**
	1. The request should include both the video file itself and user-provided metadata such as the title. This can be structured as a JSON blob containing a title and a reference to the video. The metadata is user-provided information that is separate from the inherent file metadata like content type and file extension.
5. **In a video processing system, why should the source video storage contain raw, unprocessed video rather than immediately extracting metadata and other components?**
	1. Keeping the source video storage as raw as possible maintains a clean separation of concerns. The processing service should be responsible for pulling apart the metadata, thumbnails, and separating audio/video during the processing stage. This approach ensures the source remains pristine and processing logic is centralized in the appropriate service.
