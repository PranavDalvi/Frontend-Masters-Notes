# Video Upload System Design
![](./attachments/Pasted%20image%2020260308110651.png)
1. **When designing a video upload workflow, what are key considerations regarding file specifications that need to be determined upfront?**
	1. Key file specifications include: supported video formats (e.g., MP4), maximum resolution (e.g., up to 4K), file size limits (e.g., maximum 4 gigabytes), and what type of files will be accepted. These constraints help define the scope and technical requirements of the system.
2. **What are important processing features to consider when designing a video upload service beyond just accepting the file?**
	1. Important processing features include: thumbnail generation, video preprocessing or trimming, caption/subtitle generation, audio processing (since audio tracks often need to be treated separately during video format conversion), and determining if any transcription services are needed.
3. **How can you estimate the upload capacity requirements for a video upload service?**
	1. You can estimate capacity by determining: the total number of users, the average upload frequency per user (e.g., 1 upload per day per user), and then calculating the total uploads per day. For example, 1000 users uploading once per day equals 1000 uploads per day, which helps determine system load and resource requirements.
4. **Why is it important to make explicit assumptions about audio handling when designing a video processing system?**
	1. Audio tracks often need to be treated separately when processing or converting video formats. Different video formats handle audio differently (e.g., some formats like MKV support multiple audio tracks). Making this assumption explicit helps ensure the system design accounts for the additional processing complexity and resources needed for audio handling.
5. **What types of functional and non-functional requirements should be gathered before designing a video upload workflow?**
	1. Functional requirements include supported formats, file size limits, thumbnail generation, and preprocessing needs. Non-functional requirements include performance metrics like upload speed targets, processing time constraints, concurrent user capacity, and service availability requirements. These requirements may be adjusted based on business priorities.
